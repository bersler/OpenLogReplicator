=== DML operations

Supports INSERT (including multi-row INSERT), UPDATE, and DELETE (including multi-row DELETE).
Direct-path INSERT (/*+ append */) is not supported.

=== Table characteristics

Supports nullable columns, invisible columns, default values, up to 1000 columns (till 21c) or 4095 colums (23ai), row chaining, savepoint rollbacks, partitioned tables, row dependencies, and BASICFILE LOBs.

Transactions that are rolled back are ignored.

=== Column types

Supported types include (examples): varchar2, number, date, raw, xmltype (CLOB storage), char, binary_float/double, CLOB, BLOB, timestamps, intervals, urowid, timestamp with local time zone, boolean.
Unsupported types are emitted as `"?"`.

=== DDL and schema updates

DDL is processed by parsing associated system-table DML and updating the internal schema.
DDL may span multiple transactions (e.g., table then indexes).
Temporary inconsistencies (e.g., missing index) may cause transient warnings; these can be ignored.

=== Supported character sets [[supported-character-sets]]

All source character fields are read according to the source character set.
Output is always UTF-8 (Unicode).
Character-set conversion can be disabled for testing.
See xref:../json/format.adoc#char[char].
