OpenLogReplicator advances positions only after whole LWNs (Log Writer Number blocks) are processed.
After each LWN a checkpoint record is emitted; by default checkpoint records are enabled.

Example checkpoint payload:
"payload":[{"op":"chkpt","seq":1763,"offset":15872}]

To disable checkpoint messages set the reader `flags` accordingly.

Checkpoint records include `seq` (redo file sequence), `offset` (byte offset), and optionally `"redo":true` when a redo switch occurred.
Checkpoints are useful as heartbeats — they are emitted every few seconds in a healthy stream.
