On first run, a schema snapshot for tracked schemas is created.
Only configured schemas are collected by default.

If you add schemas to the configuration after the first run, replication will not start for them until schema snapshots exist.
Recreating the schema (or removing checkpoints) may be required.

=== Schema creation

Schemas and tables to replicate are defined in the configuration; regex wildcards are supported.

If a schema does not yet exist at startup, OpenLogReplicator begins tracking it once it appears.
